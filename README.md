## 主な機能と特徴

### 1. 超高性能な逆算最適化エンジン
- **トリガー全消費ロジック**: 前半戦終了（Day 6）およびイベント最終終了（Day 13）の目標時点で、抱えているトリガーを限界（720個未満の端数）まで綺麗に消費しきれる最適な「貯め・吐き」のセット数を秒単位で全探索します。
- **当日曲の長さへの適応**: 13日間の日替わり楽曲の長さを考慮し、**「当日曲が短い日 ➔ 貯め（お仕事＋当日曲）」**、**「当日曲が長い日 ➔ 吐き（イベント曲通常・スキップ）」**を優先的に自動配分します。

### 2. 周年イベ特有の仕様・ガチ勢戦略を極力再現
- **調整日（Day 6, 13）へのスキップ最優先配分**: 時間調整が最も繊細で過酷な調整日の有効枠へ、スキップチケット（手動・オート）を最優先で集中配分。ラストスパートの事故を防ぎます。
- **ブースト＆配布トリガー自動トレース**: リフレッシュ終了時に自動付与される「ブーストアイテム（最大2個保持）」や「ログイン・デイリーミッション枠（4,000トリガー）」をタイムライン上で自動加算。ブースト中のイベント曲2倍pt（4,296pt）の最優先消化もシミュレートに組み込んでいます。
- **後半戦の元気3倍消費対応**: 後半戦（Day 7以降）に入ると、お仕事周回1回あたりに必要なセット数（30回 ➔ 20回）の変化による時間短縮を自動で反映します。

### 3. 実戦向けリアルタイム微調整 UI
- **ラップタイム自動計測＆手動微調整**: ストップウォッチ機能による「お仕事➔当日曲」の連続実測だけでなく、端末のロード差や手の疲労に合わせた1秒単位の手動直接入力（一桁目の消去ストレスなし）にも対応。
- **リフレッシュの連続8時間自動フィル**: 各日のリフレッシュ開始時間を選択するだけで、そこから連続する8時間（日跨ぎ対応）をスケジュールに自動マッピング＆編集ロック。ガシャ更新や日付更新の演出ロス（約5分）を回避する「17時以降設定」などの立ち回りを安全に計画できます。
- **タイムトラベル（仮想デバッグ）モード**: 本番期間外でも、仮想のイベント日時を操作して動作や推移をテストできるシミュレーション機能を搭載。

## 使い方
1. `index.html` をブラウザで開きます。（GitHub Pages等でのWeb公開を推奨）
2. **[現在の獲得・所持資源ステータス]** に、現在の「累積pt」「所持トリガー」「スキップチケット残数」を入力します。
3. **[ラップタイム自動計測]** または入力欄を使って、ご自身の端末環境における実測秒数をセットします。
4. **[各日当日曲の長さ設定]** に、事前に判明している楽曲秒数を入力します。
5. **[スケジュール設定]** で、各Dayの「リフレッシュ開始時間」および時間帯ごとの方針（手動周回 / オート周回 / 稼働不可）を割り当てます。
6. 最下部の **[逆算最適化アロケーション計画]** および **[精密シミュレーション・タイムライン]** に、現時点で取るべき最善の行動プラン（あと何セット貯めて、何回イベ曲を叩けばいいか）が30分刻みでリアルタイムに出力されます。

## セキュリティ・データ通信について
- **完全ローカル動作**: 本ツールは完全にユーザーのブラウザ上（JavaScript）だけで計算を行っています。外部のサーバーへプレイデータ、アカウント情報、スケジュール（生活リズム）などを送信する通信は一切行っていません。
- **自動セーブ機能**: 入力したデータや組み立てたスケジュールは、ブラウザの `LocalStorage` に自動で保存されます。次回アクセス時もそのまま前回の続きから再開可能です。
  *(※ブラウザのシークレットモード/プライベートブラウザでは保存されませんのでご注意ください)*
